# 投资委员会（IC）评分框架
## 策略优先级与资本配置决策制度

---

## 一、文档目的（Purpose）

本文件定义了投资委员会（Investment Committee, IC）  
在 **策略已通过 CRO 风险放行（Risk Clearance）之后**，  
用于进行：

- 策略优先级排序
- 资本配置决策
- 风险预算分配

的重要评分制度。

IC **不负责判断策略是否安全**，  
IC 只负责判断：

- 是否值得在“当前阶段”投入资本
- 是否优于其他已通过风险放行的策略
- 应该分配多少风险预算

---

## 二、适用前提（Preconditions）

只有满足以下全部条件的策略，  
才允许进入 IC 评分流程：

1. 已完整通过 PM / Quant / Execution / Portfolio / Stress 评审
2. 已获得 CRO 的明确 Risk Clearance
3. 风险边界已被量化或条件化描述

**未通过 CRO Risk Clearance 的策略，禁止进入 IC 评分。**

---

## 三、IC 的核心职责边界（Mandate）

IC 的职责是：

- 比较“多个都能活的策略”
- 解决资源稀缺下的选择问题
- 显性化机会成本与取舍逻辑

IC **不承担以下职责**：

- 不修改策略逻辑
- 不重新定义风险红线
- 不 override CRO
- 不因策略“看起来聪明”而破例

---

## 四、IC 策略评分表（Scoring Table）

IC 必须使用以下标准化评分表。  
每一维度独立评分，范围为 **0–10 分**。

| 评分维度 | 含义说明 | 得分（0–10） |
|---------|----------|--------------|
| Edge 强度 | 策略是否具备非共识、可持续的 alpha 来源 | |
| Regime 匹配度 | 是否契合当前市场环境与阶段判断 | |
| 风险效率 | 单位风险对应的预期回报质量 | |
| 组合互补性 | 是否降低整体组合相关性或尾部风险 | |
| 执行与容量 | 滑点、费用、规模放大能力 | |
| 压力情景表现 | 在错配或极端环境下是否可控 | |
| 操作复杂度 | 对系统、监控、人力的额外负担 | |

**总分上限：70 分**

---

## 五、评分维度解释（防止误用）

### 1. Edge 强度
- 是否依赖普遍可见信号？
- 是否容易被拥挤或复制？
- Edge 是否随时间衰减？

---

### 2. Regime 匹配度
- 当前是否是该策略的“主场”？
- 是否依赖特定波动、趋势或流动性环境？
- 是否存在明显的 regime 反转风险？

---

### 3. 风险效率
- Sharpe / IR 是否具有吸引力？
- 回撤是否与回报成比例？
- 是否存在“赚小钱、亏大钱”的结构性问题？

---

### 4. 组合互补性
- 与现有策略的相关性如何？
- 是否增加同一风险因子的集中度？
- 是否改善组合尾部结构？

---

### 5. 执行与容量
- 是否依赖极端精细的执行条件？
- 规模扩大是否显著侵蚀 alpha？
- 是否对交易所或流动性高度敏感？

---

### 6. 压力情景表现
- 在 Stress 测试中是否存在失控路径？
- 是否存在非线性、路径依赖型损失？
- 在异常行情中是否可被人工或系统干预？

---

### 7. 操作复杂度
- 是否增加系统不稳定性？
- 是否引入新的操作或合规风险？
- 是否显著提高维护成本？

---

## 六、IC 决策规则（Decision Rules）

IC 不进行“是否可行”的讨论，  
只进行“是否值得现在做”的判断。

---

### 1. GO（允许上线）

满足以下全部条件：

- 总分 ≥ 55
- 任一单项评分不得 ≤ 4
- 「执行与容量」「组合互补性」评分 ≥ 6

决策含义：

- 策略具备足够的资本效率
- 值得占用当前风险预算
- 可进入小规模或分阶段上线

附加要求：

- 明确初始风险预算
- 明确监控指标与止损升级条件

---

### 2. HOLD（暂缓执行）

满足以下任一情况：

- 总分介于 45–54
- 当前阶段存在更高优先级策略
- 与现有策略高度重叠
- Regime 匹配度不足

决策含义：

- 策略本身并非失败
- 但当前时点不值得投入资本

必须给出：

- 触发重新评估的明确条件
- 需要改进的关键维度

---

### 3. NO-GO（不执行）

满足以下任一情况：

- 总分 < 45
- 风险效率明显劣于现有策略
- 操作复杂度或机会成本不可接受

决策含义：

- 即使策略“能活”，也不值得做
- 应明确是重构还是放弃

---

## 七、治理与审计原则（Governance）

- IC 评分不是民主投票
- 分数用于：
  - 暴露分歧
  - 强制权衡
  - 支持事后复盘

每一次 IC 决策必须被记录在 `decisions/` 目录下，
并可在未来被完整追溯。

---

## 八、核心原则（Core Principle）

**生存是强制条件。  
收益吸引力是可选条件。  
资本只分配给同时满足两者的策略。**
